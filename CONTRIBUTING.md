# Contributing

Thanks for your interest in contributing! Here's how you can help.

## Ways to contribute

- **Ideas** — rhythm patterns, color palettes, visual effects
- **Code** — performance, accessibility, new features
- **Design** — UI polish, responsive tweaks
- **Bugs** — report issues or send a fix

## Development

This is a single-file app. Open `index.html` in your browser and edit. No build step, no install.

## Pull request guidelines

- Keep changes focused — one feature/fix per PR
- Test manually in Chrome, Firefox, and Safari
- Update the README if adding new controls or features
- Be kind and constructive in reviews

## Code style

- 2-space indentation
- No semicolons (unless required)
- camelCase for variables
- Meaningful names over comments

## Questions?

Open an issue — happy to help!
